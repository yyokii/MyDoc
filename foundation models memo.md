# LLM と アプリ

* 出力における特定プロパティについて、その内容が制限の厳しいものの場合調整がかなり大変なので、厳密な表現を要するものではなく「雰囲気」を説明するような仕様に落とし込むと楽。
  * たとえば、英単語の意味を [String] で出力するような仕様だと意味の厳密性や表現の仕方を制御したくなるが、その単語を説明する、ぐらいのプロパティにしておけば出力の許容できる幅が生まれてプロンプトの調整が楽。
  * 厳密な出力が必要な場合は LLM API を、そうでないならローカルLLM を使う

ねえ！ここ14日間の歩数について、ほんとにすごいね！特に10月12日から13日の間は、なんと週平均で 9347歩を達成したんだよ！ これは本当に素晴らしい達成感で、前向きなエネルギーが一気に湧いてくるよ。 それに、10月11日からの強さの回復も目覚ましい！毎週少しずつでも、進歩を重ねているその姿に、本当に感謝してる。毎日一歩ずつ、でも確実に前に進んでいることを誇りに思うね！ これからも、小さな勝利をたくさん見つけていって、一緒に楽しんでいこう！

### 結論

向いてないタスク

- ある程度“正解”が存在する分析タスク
  - 分析・比較・%改善など
- 数値を生むロジック（ルールベース）
- 医療/健康効果の断定表現

やっても良さそうなこと

- 言い回し最適化（自然で柔らかく）
- 共感・気づき（感性）
- 要約・分類（非数値）
- 多言語へのリライト

（所感)

* 禁止事項もたまに無視される印象
